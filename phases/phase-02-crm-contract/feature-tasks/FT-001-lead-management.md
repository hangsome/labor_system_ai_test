# Feature Task: FT-001-lead-management

> 阶段: Phase 02
> 模块: crm-lead
> 创建时间: 2026-02-26T21:51:26+08:00

---

## 功能概述

建设客户线索管理闭环，覆盖线索创建、状态流转、跟进记录和列表查询。目标是让业务开发可以持续跟踪转化进度，并把线索数据稳定传递给后续用工单位与合同流程。

## 需求描述

### 功能需求
1. 支持线索基础信息维护和状态流转。
2. 支持跟进记录新增与时间线查询。
3. 支持线索列表筛选、分页、详情查看。

### 非功能需求（如有）
- 性能：线索列表查询 P95 小于 300ms。
- 安全：仅允许具备权限角色访问线索接口。
- 兼容性：页面在 Chrome/Edge 最新两个版本可用。

## 技术设计

### 前端
- 页面/组件：线索列表页、线索详情页、跟进记录抽屉。
- 路由：`/crm/leads`、`/crm/leads/:id`。
- 状态管理：线索筛选条件、分页状态、详情缓存。

### 后端
- API 接口：线索 CRUD、状态流转、跟进记录。
- 服务层：状态机校验、跟进记录写入、审计事件触发。
- 数据层：`customer_lead` 及跟进相关查询。

### 数据库
- 涉及表：`customer_lead`
- 变更类型：新建

## 验收标准

### 前端验收
1. 可按状态、关键字筛选线索并分页展示。
2. 线索详情页可展示跟进记录时间线。

### 后端验收
1. 线索状态流转遵循定义规则，非法流转被拒绝。
2. 跟进记录写入后可按线索维度查询。

### 集成验收
1. 从线索创建到状态更新的链路可通过集成测试。
2. 线索核心用户故事可通过 E2E。
