# Feature Task: FT-003-contract-lifecycle

> 创建时间: 2026-02-26T21:51:26+08:00

## 功能概述

建设劳动合同生命周期管理能力，覆盖合同创建、签署、续签和终止。通过明确状态机与业务规则，为后续考勤、结算和支付阶段提供稳定合同基线。

## 需求描述

### 功能需求
1. 支持合同新增及基础字段校验。
2. 支持签署、续签、终止状态流转。
3. 支持合同列表、详情、时间线查询。

### 非功能需求（如有）
- 性能：合同列表查询 P95 小于 300ms。
- 安全：合同关键字段修改进入审计日志。
- 兼容性：合同状态变更接口保持幂等。

## 技术设计

### 前端
- 页面/组件：合同列表页、合同详情页、生命周期操作区。
- 路由：`/contract/contracts`、`/contract/contracts/:id`。
- 状态管理：合同状态过滤、详情缓存、操作反馈。

### 后端
- API 接口：合同 CRUD、签署、续签、终止。
- 服务层：生命周期状态机、合同有效期校验。
- 数据层：`labor_contract` 表及关联查询。

### 数据库
- 涉及表：`labor_contract`
- 变更类型：新建

## 验收标准

### 前端验收
1. 合同详情页可展示生命周期状态与操作入口。
2. 续签/终止动作具备确认和错误反馈。

### 后端验收
1. 合同状态机符合签署/续签/终止规则。
2. 非法状态流转返回可读错误码。

### 集成验收
1. 合同生命周期主链路可通过集成测试。
2. 从线索转化到合同签署的链路可通过 E2E。
