# Feature Task: FT-004-settlement-rule-versioning

> 阶段: Phase 02
> 模块: contract-settlement-rule
> 创建时间: 2026-02-26T21:51:26+08:00

---

## 功能概述

建设结算规则版本化管理能力，支持规则维护、生效版本控制与唯一性约束。该能力确保同一合同在同一生效区间只有唯一有效规则，降低结算计算歧义。

## 需求描述

### 功能需求
1. 支持结算规则创建、发布、停用。
2. 支持规则版本管理与历史追溯。
3. 支持规则与合同绑定关系维护。

### 非功能需求（如有）
- 性能：规则查询与版本切换操作响应稳定。
- 安全：规则发布与停用操作保留审计轨迹。
- 兼容性：规则结构变更需同步 API 与文档。

## 技术设计

### 前端
- 页面/组件：结算规则列表、规则版本面板。
- 路由：`/contract/settlement-rules`。
- 状态管理：规则版本切换状态与发布确认态。

### 后端
- API 接口：规则 CRUD、版本发布、版本查询。
- 服务层：版本唯一性校验、生效区间校验。
- 数据层：`settlement_rule` 表与索引约束。

### 数据库
- 涉及表：`settlement_rule`
- 变更类型：新建

## 验收标准

### 前端验收
1. 规则列表可展示版本并支持切换查看。
2. 发布/停用操作反馈清晰且状态一致。

### 后端验收
1. 同一合同同一生效区间仅允许一个有效规则版本。
2. 规则版本历史可追溯且不可被误覆盖。

### 集成验收
1. 合同与规则绑定后可正确查询有效版本。
2. 规则版本切换不影响历史记录查询。
