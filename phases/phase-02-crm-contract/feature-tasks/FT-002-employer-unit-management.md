# Feature Task: FT-002-employer-unit-management

> 阶段: Phase 02
> 模块: crm-employer
> 创建时间: 2026-02-26T21:51:26+08:00

---

## 功能概述

建设用工单位管理能力，支撑单位信息维护、资质状态管理与关联查询。该能力作为合同签署前置条件，保证主体信息完整且可追踪。

## 需求描述

### 功能需求
1. 支持用工单位新增、编辑、启停用。
2. 支持资质字段维护和状态展示。
3. 支持单位与线索、合同关联查询。

### 非功能需求（如有）
- 性能：单位列表查询 P95 小于 300ms。
- 安全：单位变更操作纳入审计。
- 兼容性：后端接口遵循统一错误码。

## 技术设计

### 前端
- 页面/组件：用工单位管理页、单位表单弹窗。
- 路由：`/crm/employers`。
- 状态管理：单位列表查询条件与编辑态。

### 后端
- API 接口：单位 CRUD 与状态更新。
- 服务层：资质校验、状态变更约束。
- 数据层：`employer_unit` 表读写。

### 数据库
- 涉及表：`employer_unit`
- 变更类型：新建

## 验收标准

### 前端验收
1. 用工单位列表可分页、筛选、进入编辑。
2. 用工单位状态在页面可正确展示。

### 后端验收
1. 用工单位 CRUD 能按约束完成。
2. 非法状态变更返回业务错误。

### 集成验收
1. 用工单位与合同信息可关联查询。
2. 单位到合同前置条件链路可被验证。
