# Feature Task: FT-003-audit-and-platform-base

> 阶段: Phase 01
> 模块: platform
> 创建时间: 2026-02-26T17:30:00+08:00

---

## 功能概述
建立审计日志基础能力，记录关键写操作并提供可检索能力。
为后续审查、风控和问题追踪提供统一审计底座。

## 需求描述

### 功能需求
1. 对关键写操作记录审计日志（用户、模块、动作、结果、时间）。
2. 提供审计日志查询接口。
3. 前端提供审计日志列表页面骨架和基础筛选项。

### 非功能需求（如有）
- 性能：审计写入不阻塞主交易流程。
- 安全：日志字段脱敏，避免泄漏敏感数据。
- 可维护性：日志结构统一可扩展。

## 技术设计

### 前端
- 页面/组件：审计日志列表页、筛选器组件。
- 路由：`/system/audit`。
- 状态管理：审计查询条件与分页状态。

### 后端
- API 接口：审计日志分页查询接口。
- 服务层：审计记录写入服务与查询服务。
- 数据层：`audit_log` 按时间和模块索引查询。

### 数据库
- 涉及表：`audit_log`。
- 变更类型：基线迁移补齐/索引确认。

## 验收标准

### 前端验收
1. 审计日志页面可访问并显示分页列表占位。
2. 可按模块/时间进行筛选查询。

### 后端验收
1. 关键写操作可产生审计记录。
2. 查询接口返回结构与审计字段完整。

### 集成验收
1. 执行写操作后可在审计接口查询到记录。
2. 敏感字段输出符合脱敏要求。
